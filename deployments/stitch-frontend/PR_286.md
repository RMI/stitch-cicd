## Summary

Adds a single resource view component that allows users to fetch and display individual resources by ID. This builds on the resources list view by providing detailed resource inspection capabilities.

### Key Features

- **Resource by ID**: Input field to specify resource ID with Enter key support for quick fetching
- **JSON Display**: Clean JSON visualization of resource data
- **Error Handling**: Graceful handling of 404 (Not Found) and other errors with appropriate UI feedback
- **Cache Management**: Clear cache functionality for individual resources
- **Development Tools**: Seed script to populate test data for development

### Technical Implementation

- Created reusable `JsonView` component for displaying JSON data with loading/error states
- Built `Input` component with auto-sizing, keyboard support, and accessibility features
- Implemented `ResourceView` component with ID input, fetch triggers, and cache management
- Added comprehensive test coverage for all new components
- Created `seed-resources.js` script to POST sample resources to the API for testing
- Added extensive README documentation explaining the TanStack Query key factory pattern
- Updated `App.jsx` to display both resources list and single resource views

### Files Changed

**New Components:**
